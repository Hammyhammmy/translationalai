Technical Appendix: Logic & Verification

What is "AST"?

An Abstract Syntax Tree is a structured, tree-based representation of code that captures its logical structure. Whitespace and formatting are irrelevant; only the semantic structure matters.

Logic Representation

Example Pydantic Code 
class Patient(BaseModel):
    age: int
    psa: float | None

AST Interpretation

ClassDef: Patient
Inherits: BaseModel
Fields:
• age → int
• psa → Optional[float]

Static AST Validation

We validate the structure of generated Pydantic models by inspecting their AST without executing the code. This allows us to enforce:

Layer What it checks Guards against
AST Validation Model structure, types, safety Logical flaws, unsafe code
Pydantic Validation Data values at runtime Invalid patient data, missing fields
"AST validation ensures the container is correct before data ever enters it."

Why AST validation matters

Determinism: Schema is structurally verifiable before deployment.

Auditability: Prove models conform to institutional rules.

Compatibility: Detect breaking changes structurally.

Safety: Catch syntactically valid but semantically unsafe code.

Frequently Asked Questions

What is "Structural Integrity Validation"?

Think of it as a safety blueprint check. While standard checks look at the data itself, we check the actual "machinery" doing the work. This ensures that the extraction process is structurally sound and won't produce invalid or broken data models.

Why not just run the code to see if it works?

Running code generated by an AI can be unpredictable. By using Static Validation, we can "see" what the code will do without ever actually running it. This is much safer and allows us to catch errors or security risks before they ever touch your clinical data.

How does this help my research project?

It guarantees Data Integrity. In clinical research, a small change in how data is labeled can break your entire analysis pipeline. Our system ensures that the extraction models remain perfectly consistent, even if the underlying AI updates or drifts over time.

Is this check automatic?

Yes. Every time you create or update a pipeline, the system performs these structural checks in milliseconds. You don't need to be a programmer to benefit from this; the "guardrails" are built into the platform by default.

What happens if a check fails?

If the system detects a structural flaw, it will halt the process and provide a clear explanation. This prevents "broken" data from ever reaching your database, saving you hours of manual cleanup later.
